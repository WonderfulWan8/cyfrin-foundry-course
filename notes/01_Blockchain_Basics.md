# Web3 核心术语表 (Glossary)

## 基础概念 (Basics)

### Blockchain (区块链)
* **定义**: 一个不可篡改、去中心化的分布式账本。
* **前端类比**: 一个全球共享的 Redux Store，所有节点都存有完整状态副本。任何人都可以读取 (`getState`)，但只有通过交易 (`dispatch action`) 才能修改。
* **关键特性**: Trust Minimized (最小化信任), Immutable (不可篡改)。

### Smart Contract (智能合约)
* **定义**: 部署在区块链上的自动执行程序。
* **前端类比**: 部署在区块链上的 Serverless Function (无服务器函数)。它有自己的地址，有持久化的状态 (State)，且逻辑一旦部署无法修改（通常情况）。
* **代码思维**: 后端逻辑 + 数据库存储 = 智能合约。

### Node (节点)
* **定义**: 运行区块链客户端软件的计算机。
* **前端类比**: 后端服务器实例。它们共同维护着网络的共识。我们需要连接节点 (Provider) 才能与区块链交互。

### Transaction (交易/Tx)
* **定义**: 对区块链状态的唯一修改方式。
* **前端类比**: POST 请求。每一笔交易都需要签名 (Auth) 并支付 Gas 费。

### Gas (Gas 费)
* **定义**: 也就是计算资源的费用。
* **前端类比**: AWS Lambda 的运行费用。为了防止死循环攻击，EVM 执行每一步操作（Opcode）都要收费。
* **公式**: `Gas Fee = Gas Price * Gas Used`

### Mainnet (主网)
* **定义**: 真实的区块链网络，使用真实加密货币
* **前端类比**: 生产环境 (Production Environment)。所有交易都是真实且不可逆的。
* **对比**: 测试网 (Testnet) 类似于开发环境 (Development Environment)，用于测试和调试。

### Faucet (领水)
* **定义**: 免费获取测试网代币的服务。
* **前端类比**: 类似于开发环境中的 Mock API，允许开发者获取测试资源而不花钱。
* **使用场景**: 在测试网部署和测试智能合约时使用。

### Oracle (预言机)
* **定义**: 提供链下数据的服务，允许智能合约访问外部世界的信息,把数据带上链的工具。
* **前端类比**: 类似于后端 API，智能合约通过预言机获取外部数据（如价格、天气等）。
* **使用场景**: 在 DeFi 协议中获取价格数据，或在 NFT 市场中验证资产真实性。 

### Oracle Problem (预言机问题)
* **定义**: 区块链无法直接访问链下（Off-chain）数据，但智能合约又依赖这些数据。简单理解，区块链是“瞎子”，需要张三去告诉它外面发生了什么，但是依靠单一的张三，就违背了去中心化的原则，所以需要多个张三来验证信息的真实性。
* **风险**: 如果依赖单一数据源，该数据源就成为单点故障（Single Point of Failure），破坏了去中心化。
* **解决**: 使用去中心化预言机网络 (如 Chainlink)，通过多节点验证和经济激励（Staking）来保证数据的真实性。

### Decentralized Oracle (去中心化预言机)
* **定义**: 由多个独立节点组成的预言机网络，提供去中心化的数据源。
* **前端类比**: 类似于分布式缓存系统，多个节点共同维护数据的准确性和可用性。
* **使用场景**: 在需要高可靠性和去中心化的数据源时使用，如 DeFi 协议中的价格预言机。

### Hybrid Smart Contract (混合智能合约)
* **定义**: 结合了链上代码（On-chain Logic）和链下数据/计算（Off-chain Data/Compute）的智能合约。
* **结构**: 
  * **On-chain**: 负责持有资金、状态管理和最终结算。
  * **Off-chain (Oracle)**: 负责提供真实世界数据、外部API连接或复杂计算（如随机数生成）。
* **价值**: 突破了区块链“无法主动获取外部数据”的限制，极大地扩展了应用场景。

### Chainlink
* **定义**: 一个去中心化的预言机网络，提供可靠的链上和链下数据连接。
* **前端类比**: 类似于前端的 API Gateway，提供统一的接口来访问各种数据源。
* **使用场景**: 在 DeFi 协议中获取价格数据，或在 NFT 市场中验证资产真实性。

### EVM (Ethereum Virtual Machine)
* **定义**: 以太坊虚拟机，是智能合约的运行环境。
* **前端类比**: 类似于浏览器的 JavaScript 引擎，但它执行的是智能合约代码 (Bytecode)。
* **代码思维**: 前端的 React/Vue 对应智能合约的 Solidity。
* **价值**: 跨平台兼容性，任何支持 EVM 的区块链都能运行相同的智能合约代码。

### EVM equivalent Chains (EVM 兼容链)
* **定义**: 支持 EVM 的区块链网络，如 Binance Smart Chain、 Polygon、Avalanche 等。
* **前端类比**: 类似于不同浏览器（Chrome、Firefox、Safari）都能运行相同的 JavaScript 代码。
* **价值**: 开发者可以轻松地将智能合约从以太坊迁移到其他 EVM 兼容链，扩大用户基础和应用场景。

### EVM Non-equivalent Chains (非 EVM 兼容链)
* **定义**: 不支持 EVM 的区块链网络，如 Solana、Cardano、Polkadot 等。
* **前端类比**: 类似于不同编程语言（JavaScript、Python、Ruby）有不同的运行时环境和语法。
* **挑战**: 开发者需要学习新的编程语言和工具链，增加了开发难度和成本。
* **价值**: 提供了不同的设计理念和技术创新，推动区块链生态的发展。

### EVM Compatibility Layer (EVM 兼容层)
* **定义**: 允许非 EVM 兼容链运行 EVM 智能合约的技术层。
* **前端类比**: 类似于跨平台框架（如 React Native、Flutter），允许开发者使用相同的代码库在不同平台上运行应用。
* **价值**: 降低了开发成本，促进了跨链互操作性。

### Upgradeable Smart Contract (可升级智能合约)
* **定义**: 允许在不改变合约地址的情况下更新合约逻辑的智能合约。
* **前端类比**: 类似于前端的热更新 (Hot Module Replacement)，允许在不刷新页面的情况下更新代码。
* **实现方式**: 通过代理合约 (Proxy Contract) 模式，将逻辑和数据分离，逻辑合约可以被替换，而数据合约保持不变。
* **价值**: 提供了灵活性和可维护性，允许修复漏洞和添加新功能，而不影响现有用户和数据。

### 51% Attack (51% 攻击)
* **定义**: 当单一实体控制了区块链网络超过 51%的算力或权益时，可能会对网络进行恶意操作。
* **风险**: 双重支付 (Double Spending)、阻止交易确认 (Transaction Censorship)。
* **防御**: 增加网络的去中心化程度，采用更安全的共识机制（如 PoS）。

### wallet (钱包)
* **定义**: 用于管理加密货币资产和与区块链交互的工具。
* **前端类比**: 类似于浏览器的 Cookie 管理器，但它管理的是加密货币账户和签名交易。
* **用户交互**: 用户通过钱包连接 DApp 并签名交易。

### MetaMask
* **定义**: 一个浏览器扩展钱包，用于管理以太坊账户和与 DApp 交互。
* **用户交互**: 用户通过 MetaMask 连接 DApp 并签名交易。

### Seed Phrase (助记词)
* **定义**: 一组单词，用于生成和恢复加密货币钱包。
* **前端类比**: 类似于密码管理器中的主密码，用于恢复访问权限。
* **安全性**: 助记词必须妥善保管，任何拥有助记词的人都可以访问钱包中的资产。

### Sepolia (赛波利亚测试网)
* **定义**: 以太坊的一个测试网络，用于开发和测试智能合约。
* **前端类比**: 类似于开发环境 (Development Environment)，允许开发者在不影响主网的情况下进行测试和调试。
* **使用场景**: 在 Sepolia 测试网部署和测试智能合约时使用。

### Blockchain Explorer (区块链浏览器)
* **定义**: 一个在线工具，用于查看区块链上的交易、区块和地址信息。
* **前端类比**: 类似于浏览器的开发者工具 (Developer Tools)，允许用户查看和调试区块链数据。
* **常见工具**: Etherscan、Blockchair、Ethplorer。

### Etherscan
* **定义**: 以太坊区块链浏览器，提供交易、区块和地址信息的查询服务。此处注意，如果是测试网领水测试币，需要使用对应的测试网浏览器，如 Sepolia Etherscan，注意浏览器前缀。
* **前端类比**: 类似于浏览器的开发者工具 (Developer Tools)，允许用户查看和调试区块链数据。
* **使用场景**: 用于查询以太坊网络上的交易状态、合约代码和代币信息。

### Transaction Fee (交易费)
* **定义**: 用户为在区块链上执行交易或智能合约操作支付的费用。
* **前端类比**: 类似于 API 请求的费用，用户需要为每次请求支付一定的费用以防止滥用。
* **计算方式**: Transaction Fee = Gas Used * Gas Price。

## Gas (燃气/工作量)
* **定义**: Gas是计量单位（类似于电表走的度数或者汽车耗的燃油量）。本质上是EVM执行代码所需的计算资源总和。

### Gas Price (Gas 价格)
* **定义**: 用户愿意为每单位 Gas 支付的价格，通常以 Gwei 为单位。
* **前端类比**: 类似于云计算中的按需付费，用户可以根据网络拥堵情况调整 Gas 价格以加快交易确认速度，类似于油价，高峰期加价，取决于网络拥堵程度。
* **影响因素**: 网络拥堵程度、交易优先级。

### Validator (验证者)
* **定义**: 在 PoS (权益证明) 机制下负责打包交易、执行智能合约代码并维护账本的节点。它们通过质押 ETH (32枚) 获得记账权，彻底取代了 PoW 时期的“矿工 (Miner)”。
* **类比**: 你的 **“云服务器”兼“收银员”**。你支付 Gas 费租用他们的 CPU/硬盘资源来运行代码，他们负责处理并在账本上结算。
* **价值**: 网络的执行引擎。除了赚取 Gas 费，他们还拥有决定交易排序的“上帝权力” (即提取 MEV 的能力)。

### Private Key (私钥)
* **定义**: 用于对交易进行数字签名的密钥。
* **类比**: 银行卡密码 + 银行卡本身 + 你的指纹。
* **安全性**: 私钥必须妥善保管，任何拥有私钥的人都可以控制钱包

### Public Key (公钥)
* **定义**: 与私钥配对的密钥，用于验证交易签名。
* **类比**: 银行卡号或者是收款二维码。
* **安全性**: 公钥可以公开分享，用于验证交易的合法性。

### melpool (交易池)
* **定义**: 节点收集并等待打包的交易队列。
 **前端类比**: 类似于银行的交易队列，交易需要等待被处理。
 **影响因素**: 网络拥堵程度、交易优先级。

### hard fork (硬分叉)
* **定义**: 区块链网络发生重大协议升级，导致新旧版本的区块链网络分裂。
旧版本的区块链网络继续运行，而新版本的区块链网络则从一个不同的起点开始运行。
**例子**: 2016年以太坊硬分叉，导致以太坊从 Ethereum Classic (ETC) 分离出来。

### EIP (Ethereum Improvement Proposal)
* **定义**: 以太坊改进提案，用于提出和讨论对以太坊协议进行改进的提案。EIP 1559 是一个著名的 EIP，引入了 Gas 费的新机制。

### ERC (Ethereum Request for Comment)
* **定义**: 以太坊请求评论，用于提出对以太坊协议进行改进的提案。ERC 20 是一个著名的 ERC，定义了代币的标准。
* **例子**: ERC 721 是一个著名的 ERC，定义了 NFT 的标准。

### Turing Completeness (图灵完整性)
* **定义**: 智能合约能够执行任意复杂计算的能力，类似于图灵机的能力。EVM 的图灵完整性使得智能合约能够执行复杂的逻辑，包括循环、条件判断等
* **价值**: 智能合约能够执行复杂的逻辑，包括循环、条件判断等。

### Type 0 Transaction (Type 0 交易)
* **定义**: 传统的以太坊交易类型，包含数据和 Gas 费用。

### Type 2 Transaction (Type 2 交易)
* **定义**: EIP-1559 引入的交易类型，包含基础费用和优先费用。

### Wei (微单位)
* **定义**: 以太坊的最小单位，1 ETH = 10^18 Wei。
* **前端类比**: 类似于美元中的分 (Cent)，用于精确表示小额金额。

### Gwei (吉威)
* **定义**: 以太坊中常用的单位，1 Gwei = 10^9 Wei。
* **前端类比**: 类似于美元中的分 (Cent)，用于表示 Gas 价格。

### Account Abstraction (账户抽象)
* **定义**: 将区块链账户从单纯的“私钥控制”升级为“智能合约控制”。它允许通过代码逻辑（如社交恢复、代付 Gas）来管理资产，而不再是“一把钥匙（私钥）定生死”。
* **例子 (Correct)**: Argent 钱包 或 社交恢复功能。
* **对比**: MetaMask 丢了助记词就完了；
* **AA钱包**: 丢了手机没事，可以让 3 个朋友（监护人）帮你重置访问权限，或者让应用方帮你付 Gas 费。

### Token (代币)
* **定义**: 运行在现有区块链（如以太坊）上的智能合约。它不是区块链底层的原生货币（如 ETH），而是由开发者编写的一段代码，用来定义一种新的资产。本质上，它就是一个去中心化的记账本（Excel 表格），记录了“谁拥有多少个积分”。
* **例子**: USDT 或 UNI。
* **对比**: ETH 是“原生币” (Coin)，用来支付 Gas 费；Token 是“寄生”在以太坊上的合约资产。
* **原理**: 当你转账 100 USDT 时，并没有任何物理或数字文件发生了移动。实际上是你调用了 USDT 的合约函数，让它把你的余额数字减 100，把对方的余额数字加 100。

### RWA (Real World Assets - 现实世界资产)
* **定义**: 将传统金融体系或物理世界中的资产（如房产、国债、黄金、股票），通过代币化 (Tokenization) 的方式映射到区块链上。
* **本质**: RWA = 链下资产 + 法律确权 + 链上 ERC-20 代币。它打通了 Web2（传统金融）和 Web3（去中心化金融）的隔阂。
* **例子**: USDT / USDC: 这是最成功的 RWA。它们把“美元”（现实资产）映射成了链上的 Token。
* **链上国债 (BlackRock BUIDL)**: 贝莱德把美国国债做成了 Token。持有这个 Token，就像持有国债一样能拿利息，但你可以 24 小时随时在链上转账。
* **房地产**: 一套 1000 万的房子，切分成 100 万个 Token。你花 10 块钱就能买这套房子百万分之一的所有权（收租金）。
* **总结**: RWA 就是把现实世界里值钱的东西，做成 ERC-20 代币，让它们能在区块链上跑起来。

### Stablecoins (稳定币)
* **定义**: 一种旨在保持价值稳定的加密货币。它的价格通常锚定（Peg）某种法币（如美元）或商品（如黄金），通常维持 1 Token = $1 USD。
* **作用**: 解决了比特币和以太坊价格剧烈波动的问题，使其能作为“记账单位”和“交易媒介”。它是 Web3 世界里的现金。

### DEX (Decentralized Exchange / 去中心化交易所)
* **定义**: 一个没有后端服务器、没有数据库、完全运行在区块链上的智能合约集合。它是一个点对点（P2P）的交易市场。
* **通俗理解**: 无人值守的自动兑换机。
* **CEX (如币安/Coinbase)**: 像是银行柜台。你把钱给柜员（充值），柜员在他的 Excel 表里给你记账。交易是在他们的私有服务器里完成的。如果柜员跑路，你的钱就没了。
* **DEX**: 像是代码写死的自动贩卖机。你不需要注册账号，不需要上传身份证。你连接钱包，一手交钱（Token A），代码一手交货（Token B）。资金始终在你手里，直到交换的那一秒。

### Uniswap
* **定义**: 以太坊上最著名、交易量最大的 DEX 协议。它是 DEX 领域的“开山鼻祖”和标准制定者。
* **核心机制**: AMM (自动做市商)。
* **传统模式 (Order Book)**: 买家挂单 "我要 100 块买"，卖家挂单 "我要 102 块卖"，系统撮合。这在链上太慢太贵了。
* **Uniswap 模式 (Liquidity Pool)**: 资金池。
* **池子**: 想象一个大桶，里面混着装了 ETH 和 USDT。
* **LP (流动性提供者)**: 任何人（比如你）都可以把闲置的 ETH 和 USDT 扔进桶里，成为“股东”，赚取交易手续费。
* **交易**: 当你想买 ETH 时，你往桶里扔 USDT，拿走 ETH。
定价: 由数学公式 x * y = k 自动决定。桶里的 ETH 越少，价格就自动飙升。不需要人工挂单，算法就是定价员。

### L1s (Layer 1) - 结算层 (The Settlement Layer)
* **定义**: 拥有独立共识机制和安全性的基础区块链网络（如 Ethereum, Bitcoin, Solana）。
* **核心职责**: **去中心化** 和 **安全性**。它们是最终的“真理法院”，负责确权。
* **代价**: 为了安全，必须牺牲速度，导致 Gas 昂贵。
> **代码思维**: 类似于 **后端服务器/数据库 (Backend/Database)**。它是状态的最终存储地，所有数据以它为准。

### L2s (Layer 2) - 执行层 (The Execution Layer)
* **定义**: 建立在 L1 之上的扩展解决方案（如 Optimism, Arbitrum, Base）。
* **核心机制**: **Rollups (打包)**。在链下执行成百上千笔交易，压缩成一个“证明”提交给 L1。
* **价值**: **继承 L1 的安全性**（这是关键！），同时提供极低的 Gas 和极快的 TPS。
> **代码思维**: 类似于 **压缩算法 + 批处理脚本 (Batch Processing)**。它在本地疯狂计算，只把结果哈希上传到云端。

### 🚨 关键区分：L2 vs Sidechain (侧链)
* **L2 (Rollup)**: 如 Arbitrum, Optimism。**数据发回以太坊**，安全由以太坊保障。
* **Sidechain**: 如 Polygon PoS, Gnosis。**有自己独立的验证节点**。如果侧链节点作恶，L1 救不了你。

### L2 Maturity Stages (L2 成熟度分级) - by Vitalik
* **Stage 0 (Full Training Wheels / 全辅助轮)**
  * **现状**: 官方拥有绝对控制权（很多早期的 L2 都在这里）。
  * **风险**: 既然是官方控制，那如果服务器宕机，或者官方想回滚交易，完全是他们说了算。
  * **通俗理解**: 名为区块链，实为“腾讯云数据库”。
* **Stage 1 (Limited Training Wheels / 半辅助轮)**
  * **现状**: 哪怕官方作恶，用户也有机会通过“欺诈证明”来提款。
  * **保留权限**: 官方保留了一个“紧急按钮” (Security Council)。
  * **特点**: 平时按规则运行，但出大事了（比如合约有严重漏洞），官方依然有权插手干预。目前主流 L2 (Arbitrum/Optimism) 大多处于此阶段。
* **Stage 2 (No Training Wheels / 无辅助轮)**
  * **现状**: 完全去中心化。
  * **原则**: **Code is Law (代码即法律)**。
  * **终极目标**: 哪怕官方团队解散、丢了私钥，也无法干预网络运行。这才是真正的 Web3。
  
# Web3 跨链机制核心：Bridge, Mint, & Burn
> **核心误区纠正**：资产从未真正“移动”。它们是被**锁定**在原链，并在目标链**映射**出影子。
### 1. Bridge (桥接) - 那个“柜台”
* **定义**：连接两个独立区块链（如 Ethereum 和 ZKsync）的通信协议。
* **本质**：它不是运输管道，而是一套**智能合约**。
* **职能**：负责记账和验证。它确保“这边锁多少，那边印多少”，保证总量守恒。
* **风险**：它是黑客最眼馋的攻击目标。

### 2. Mint (铸造) - 入金 (L1 -> L2)
当你把 ETH 从主网充值到 L2 时发生的动作。
* **流程**：
    1.  **Lock (锁定)**：你的真金白银（ETH）被发送到 L1 的桥接合约中，被**冻结**。谁也拿不走。
    2.  **Verify (验证)**：L2 的合约监测到“L1 已锁币”的信号。
    3.  **Mint (铸造)**：L2 合约凭空**印出**等额的代币（xETH）发送给你。
* **类比**：你去赌场柜台，把人民币锁进保险柜，柜台给你一堆塑料筹码。
* **结论**：你在 L2 上手里的币，本质上是 L1 资产的**兑换券**。

### 3. Burn (销毁) - 出金 (L2 -> L1)
当你把 ETH 从 L2 提现回主网时发生的动作。
* **流程**：
    1.  **Burn (销毁)**：你把 L2 的代币发回桥接合约。合约直接将其**销毁**（从总账本中永久删除）。
    2.  **Verify (验证)**：L1 的合约监测到“L2 已销毁”的信号。
    3.  **Unlock (解锁)**：L1 合约打开保险柜，把属于你的真金白银（ETH）**释放**回你的钱包。
* **类比**：你离开赌场，把塑料筹码扔进火炉烧掉，柜台把人民币还给你。
* **关键**：如果不销毁 L2 的影子，就无法释放 L1 的真身。

**为什么桥接最危险？**
如果 L1 的桥接合约（保险柜）被黑客攻破，里面的真 ETH 被盗光了：
* 你在 L2 上的所有代币瞬间归零。
* 因为它们背后的**抵押物**没了。
* 你手里剩下的只是一堆无法兑现的废纸（数字）。
**Rule of Thumb:** 大额资金尽量存储在 L1，L2 只放用于交互和开发的流动资金。

## 业务场景 (Use Cases - 来自课程 Lesson 4)

### DeFi (Decentralized Finance)
* **核心**: 也就是去中心化金融。
* **例子**: 不需要银行的借贷 (Aave)，不需要做市商的交易所 (Uniswap)。
* **价值**: 消除中介 (Removing the Middleman)，代码即信任。

### DAO (Decentralized Autonomous Organization)
* **核心**: 去中心化自治组织。
* **类比**: 一个由代码管理的“公司”。股东投票不是举手，而是用 Token 签名。

### DAP (Decentralized Application)
* **核心**: 去中心化应用。
* **类比**: 一个运行在区块链上的前端应用，用户通过钱包连接到智能合约。

### DEX (Decentralized Exchange)
* **核心**: 去中心化交易所。
* **类比**: 类似于传统的股票交易所，但没有中心化的中介，交易通过智能合约自动执行。

### AMM (Automated Market Maker)
* **核心**: 自动化做市商。
* **类比**: 类似于传统交易所中的做市商，但由智能合约自动执行，提供流动性池 (Liquidity Pool) 供用户交易

### NFT (Non-Fungible Token)
* **核心**: 非同质化代币。
* **类比**: 数字艺术品的所有权证明。每个 NFT 都是独一无二的，不能互换。

### Oracle (预言机)
* **核心**: 提供链外数据的服务。
* **类比**: 类似于后端 API，智能合约通过预言机获取外部世界的信息（如价格数据）。

### DeFi Staking
* **核心**: 去中心化金融质押。
* **类比**: 类似于传统银行的定期存款，但用户将资金质押到 DeFi 协议中以获得收益。

### Staking (质押/理财)
* **本质**: 将代币锁定在智能合约中以获取收益。
* **类型**:
  * **ETH Staking**: 维护公链安全（类似于银行原始股）。
  * **Lending (Yield Farming)**: 借贷生息（类似于 P2P）。
  * **Liquidity Mining (Uniswap)**: 提供交易流动性赚手续费（类似于外汇柜台）。
  * **Utility Staking (Chainlink)**: 为服务做担保（类似于保证金）。

### Single Asset Staking (单币质押)
* **核心**: 只质押一种加密货币以获得奖励。
* **类比**: 类似于将钱存入银行的储蓄账户，赚取利息。

### Liquidity Mining (流动性挖矿)
* **核心**: 通过向流动性池提供资金来赚取奖励。
* **类比**: 类似于成为市场的做市商，赚取交易费用和奖励。

---## 工具与技术 (Tools & Technologies)
### EVM (Ethereum Virtual Machine)
* **定义**: 以太坊虚拟机，是智能合约的运行环境。
* **前端类比**: 类似于浏览器的 JavaScript 引擎，但它执行的是智能合约代码 (Bytecode)。

### Solidity
* **定义**: 以太坊智能合约的主要编程语言。
* **前端类比**: 类似于 JavaScript，但专为编写智能合约设计，具有静态类型和合约特有的语法。
* **代码思维**: 前端的 React/Vue 对应智能合约的 Solidity。

### Foundry
* **定义**: 一个用于以太坊智能合约开发的工具链。
* **前端类比**: 类似于前端的 Vite 或 Create React App，提供编译、测试和部署功能。

### Hardhat
* **定义**: 另一个流行的以太坊开发环境。
* **前端类比**: 类似于前端的 Webpack，提供本地开发网络和调试工具。

### Metamask
* **定义**: 一个浏览器扩展钱包，用于管理以太坊账户和与 DApp 交互。
* **前端类比**: 类似于浏览器的 Cookie 管理器，但它管理的是加密货币账户和签名交易。
* **用户交互**: 用户通过 Metamask 连接 DApp 并签名交易。

### Infura / Alchemy
* **定义**: 提供区块链节点访问的云服务。
* **前端类比**: 类似于后端的 API 服务，开发者可以通过它们连接到以太坊网络，而无需自己运行节点。
* **使用场景**: 读取区块链数据或发送交易。

---## 进阶概念 (Advanced Concepts)
### Consensus Mechanism (共识机制)
* **定义**: 确保区块链网络中所有节点就交易顺序达成一致的协议。
* **前端类比**: 类似于分布式系统中的一致性算法（如 Paxos 或 Raft），确保所有服务器对状态有相同的视图。
* **常见类型**: PoW (工作量证明), PoS (权益证明)。
* **代码思维**: 类似于前端的状态管理库（如 Redux），确保所有组件对状态有一致的理解。
* **价值**: 去中心化、安全性。

### Sharding (分片)
* **定义**: 一种扩展技术，通过将区块链数据分割成多个部分（分片）来提高吞吐量。
* **前端类比**: 类似于数据库的分区 (Partitioning)，将数据分散存储以提高访问速度。
* **代码思维**: 类似于前端的代码拆分 (Code Splitting)，按需加载资源以提升性能。
* **价值**: 扩展性、性能提升。

### zk-SNARKs (零知识简洁非交互式论证)
* **定义**: 一种密码学技术，允许一方证明其知道某个秘密，而无需透露该秘密本身。
* **前端类比**: 类似于前端的身份验证 (Authentication)，但不需要透露密码，只需证明身份。
* **代码思维**: 类似于前端的 OAuth 流程，验证用户身份而不暴露敏感信息。
* **价值**: 隐私保护、安全性。

### Flash Loan (闪电贷)
* **定义**: 一种无需抵押即可借贷的机制，只要在同一交易中归还即可。
* **前端类比**: 类似于前端的临时状态 (Temporary State)，在一个操作中创建和销毁数据。
* **代码思维**: 类似于前端的事务 (Transaction)，确保一系列操作要么全部成功，要么全部失败。
* **价值**: 高效利用资金、套利机会
